# Review Point Guidelines

Use this file as the in-skill reference for how rich each 「復習ポイント」 section should be.

## Goal

Each review point should let the learner recall the core idea without reopening a textbook.

## Recommended Structure

For every problem, include a short 「復習ポイント」 subsection with the following as needed:

1. Definition
State the theorem, distribution, estimator, or testing setup in precise words.

2. Intuition
Explain what the concept is doing and why it matters in this problem.

3. Procedure
Summarize the solution flow in 2-4 concrete steps.

4. Pitfalls
Name common mistakes, such as parameter confusion, sign errors, or misuse of asymptotic approximations.

5. Connections
Link the problem to adjacent topics such as likelihood, sufficiency, CLT, hypothesis testing, regression, stochastic processes, or Bayesian reasoning.

## Style

- Prefer bullets over dense paragraphs.
- Keep notation consistent with the problem statement.
- If a formula is central, show it explicitly.
- If a visual interpretation matters, describe the graph or shape in words even when a figure is included.
- Do not use boxed callouts. Plain headings and lists are sufficient.
